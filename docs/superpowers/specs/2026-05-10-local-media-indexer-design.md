# 本地影视索引与分类软件设计

日期：2026-05-10

## 目标

构建一个 Windows 个人本地影视管理器，用来解决多硬盘、多目录下影视文件难以查找、分类、识别和整理的问题。第一版重点不是做播放器，也不是追求所有影片自动识别准确，而是建立可靠的本地索引、层级 TAG 分类、待确认工作台、成人库强隔离和安全文件整理流程。

## 产品边界

第一版包含：

- 扫描用户授权的普通目录和成人目录。
- 支持普通硬盘、移动硬盘和离线硬盘索引。
- 支持按硬盘、目录、TAG、导演、年代、国家、文件名等维度查找影片。
- 普通库和成人库强隔离，成人库需要软件内密码进入。
- 每个影片至少保留本地基础信息：文件名、路径、硬盘、大小、时长、分辨率、截图封面。
- 自动资料补全作为增强能力，失败不阻塞入库。
- 未识别或低置信度影片进入待确认工作台。
- 用户可填写标题、番号、网址、演员、导演、TAG 等线索。
- 可配置 API Key，让 AI 基于文本线索辅助分类、补全 TAG、生成整理建议。
- 成人库默认不上传截图、视频、音频给云端 AI。
- 支持封面/截图预览，并可用系统默认播放器打开影片。
- 支持手动确认后的复制、剪切、移动、重命名和批量整理。

第一版不包含：

- 不内置完整播放器。
- 不做云同步。
- 不做多人账号系统。
- 不做无人确认的自动移动或自动删除。
- 不把成人截图默认发送给云端模型。
- 不承诺自动资料 100% 准确，所有不确定结果都必须人工确认。

## 推荐技术栈

- 桌面端：C# / .NET 8
- UI：WPF
- 数据库：SQLite + FTS 全文索引
- 视频信息：FFmpeg / ffprobe
- 后台任务：.NET 后台任务队列
- 文件操作：Windows 原生文件 API
- AI：用户自配 API Key，第一版只发送文本线索

选择理由：软件只考虑 Windows，.NET 对 Windows 文件系统、路径处理、桌面 UI、安装包和长期维护更稳定。相比 Python 打包或跨平台框架，后期更容易维护和找人接手。

## 核心模块

- 媒体库模块：管理普通库、成人库和离线硬盘库。
- 扫描索引模块：扫描视频文件，提取路径、大小、修改时间、硬盘 ID、文件指纹。
- 视频分析模块：调用 ffprobe 读取时长、分辨率、编码，并用 ffmpeg 生成本地截图封面。
- 搜索模块：使用 SQLite FTS 支持文件名、标题、演员、导演、TAG、番号、路径搜索。
- 层级 TAG 模块：管理内置 TAG、用户自定义 TAG、多语言显示名、别名、父子关系和组合筛选。
- 资料补全模块：根据标题、年份、番号、网址、NFO 等线索抓取资料。
- AI 分类助手：只基于文本线索生成分类、TAG、重命名和整理路径建议。
- 待确认工作台：处理识别失败、低置信度匹配、AI 建议、重复项和人工补充。
- 文件操作模块：复制、剪切、移动、重命名，全部需要确认并记录日志。
- 成人库安全模块：密码解锁、独立数据库、独立封面缓存、独立搜索索引。
- 设置与权限模块：管理扫描目录、联网权限、AI Key、成人库密码、收录规则和危险操作确认。

## 数据流

普通库数据流：

```text
用户添加授权目录
-> 扫描索引模块发现视频
-> 收录规则过滤短视频、缓存、游戏素材
-> 视频分析模块生成基础资料和截图封面
-> 资料补全模块尝试匹配
-> AI 分类助手在授权后生成文本建议
-> 低置信度项目进入待确认工作台
-> 用户确认
-> 写入普通库、TAG 索引和全文搜索索引
```

成人库数据流：

```text
用户输入成人库密码
-> 解锁成人库
-> 扫描授权的成人目录
-> 本地提取文件名、路径、番号、时长、分辨率、截图封面
-> 成人资料源或用户网址补全
-> AI 只处理文本线索
-> 用户确认
-> 写入 adult.db、成人 TAG 和成人搜索索引
```

## 索引与浏览

软件提供多个入口，而不是只有一个搜索框：

- 按硬盘浏览：例如点击 D 盘，查看 D 盘已索引视频；移动硬盘离线时仍可浏览历史索引。
- 按目录浏览：保留真实文件夹层级，便于定位原始文件。
- 按 TAG 浏览：点击影片详情里的国家、导演、年代、类型、存储位置等 TAG，进入对应索引结果。
- 按条件筛选：导演、演员、国家/地区、年代、类型、评分、时长、分辨率、字幕、是否离线硬盘。
- 全文搜索：支持不完整片名、导演名、演员名、番号、路径片段、TAG key、TAG 显示名和别名。
- 组合搜索：例如 `苏联 + 战争 + 1970年代 + D盘 + 时长>20分钟`。

示例操作：

```text
媒体库 -> 按硬盘浏览 -> D盘 -> 视频列表
影片详情 -> 点击“苏联” -> 所有 country.soviet_union 影片
TAG索引 -> 选择 country.soviet_union + genre.war + decade.1970s
搜索框 -> 输入导演名或路径片段 -> 缩小范围
```

## 收录规则

默认不把所有视频文件都当作影视作品。第一版使用候选收录规则：

- 视频时长大于默认阈值 20 分钟。
- 文件体积大于默认阈值 100 MB。
- 文件位于用户授权的媒体目录内。
- 扩展名属于常见视频格式，例如 mkv、mp4、avi、mov、wmv、flv、ts、m2ts。
- 排除明显缓存目录、临时目录、游戏安装目录、浏览器缓存目录。

低于阈值或不确定的视频进入“短视频/待判断”，用户可以选择：

- 忽略。
- 标记为影视作品。
- 标记为短片。
- 标记为游戏过场动画。
- 永久排除此目录。
- 为某个目录单独设置收录规则。

## 层级 TAG 系统

TAG 是核心索引模型。底层保存稳定英文 key，界面显示支持中文、英文和别名。

示例：

```text
country
country.soviet_union
country.japan
person.director.akira_kurosawa
person.actor.toshiro_mifune
decade.1970s
genre.war
genre.history
library.normal
library.adult
storage.drive.d
storage.offline
status.unconfirmed
```

中文界面可显示为：

```text
国家 / 苏联
国家 / 日本
导演 / 黑泽明
演员 / 三船敏郎
年代 / 1970年代
类型 / 战争
存储 / D盘
状态 / 待确认
```

第一版内置 TAG 根节点：

```text
country.*
language.*
decade.*
year.*
genre.*
topic.*
person.director.*
person.actor.*
series.*
studio.*
rating.*
quality.*
subtitle.*
library.*
storage.*
status.*
custom.*
```

成人库可使用独立 TAG 根节点：

```text
adult.code.*
adult.actor.*
adult.studio.*
adult.series.*
adult.topic.*
```

普通模式不加载 adult.* TAG。

TAG 管理能力：

- 创建 TAG。
- 合并 TAG。
- 修改显示名和别名。
- 设置父子关系。
- 给影片批量添加或移除 TAG。
- 点击 TAG 查看关联影片。
- 多 TAG 组合筛选。
- AI 生成 TAG 建议，用户确认后写入。
- 设置目录规则，例如某目录下的影片自动添加 `country.soviet_union`。

## 多语言与本地化

第一版界面以中文为主，但必须预留快速本地化口子，避免后续把 UI 字符串散落在 XAML 和业务代码里。

本地化原则：

- 所有 UI 固定文案通过资源 key 获取，不直接硬编码在视图或业务逻辑中。
- 第一版内置 `zh-CN` 和 `en-US` 两套资源文件，中文资源完整，英文资源可以先覆盖主要入口和按钮。
- TAG 底层 key 始终使用稳定英文，例如 `country.soviet_union`，不随界面语言变化。
- TAG 显示名支持多语言字段和别名，例如中文显示“苏联”，英文显示“Soviet Union”。
- 搜索索引同时包含 TAG key、当前语言显示名、其他语言显示名和别名。
- 用户自定义 TAG 至少保存默认显示名，未来可补充多语言显示名。
- 语言设置保存在本地配置中，切换语言后重新加载界面资源，不改变数据库中的稳定 key。

第一版需要实现的本地化口子：

- 一个 `Localization` 服务，用资源 key 返回当前语言文本。
- 资源文件目录，例如 `Resources/Strings.zh-CN.json` 和 `Resources/Strings.en-US.json`。
- UI 文案统一使用资源 key。
- 设置页保留“界面语言”选项。
- TAG 显示逻辑通过当前语言优先选择显示名，找不到时回退到英文名，再回退到 TAG key。

## 资料补全与人工补充

资料补全分为五层：

```text
1. 文件名/路径解析
2. 在线资料源搜索
3. 用户补充线索
4. AI 文本分类助手
5. 人工确认入库
```

文件名/路径解析提取：

- 片名候选
- 年份
- 国家/地区词
- 清晰度
- 字幕信息
- 剧集季集号
- 成人番号
- 系列名
- 文件夹名中的导演、演员、合集名

普通库资料源：

- TMDb
- OMDb / IMDb 关联信息
- 本地 NFO 文件
- 用户提供的网址

成人库资料源：

- 番号、标题、演员相关资料源
- 本地 NFO 文件
- 用户提供的网址
- 后续可通过插件添加资源站适配器

识别失败或低置信度项目进入待确认工作台。用户可补充：

```text
标题
原名
年份
国家/地区
导演
演员
番号
系列
介绍网址
备注
TAG
是否成人
```

## AI 分类助手

AI 的定位是分类助手，不是成人内容视觉识别器。第一版默认只发送文本，不发送截图、视频或音频。

AI 输入：

```text
文件名
文件夹名
已解析字段
用户填写线索
网页标题
网页摘要
本地 NFO 文本
已有 TAG
时长、分辨率、文件大小
```

AI 输出：

```text
是否应收录
普通/成人/待确认
影片类型
国家/地区
年代
导演/演员候选
TAG 建议
简介摘要
重命名建议
整理目录建议
重复项风险
置信度
需要人工确认的问题
```

AI 不直接修改数据库，只生成建议。用户确认后才写入。

成人库 AI 边界：

- 不默认上传截图。
- 不上传视频。
- 不上传音频。
- 不要求模型生成露骨内容描述。
- 只做中性分类、资料摘要和 TAG 建议。
- 云端 AI 拒绝、超时或失败时，不影响本地工作台继续使用。
- 未来可预留本地模型接口，但第一版不强依赖。

## 成人库强隔离

成人库按强隔离设计，不只是隐藏入口。

存储隔离：

```text
普通库数据库：normal.db
成人库数据库：adult.db
普通封面缓存：cache/normal/
成人封面缓存：cache/adult/
普通搜索索引：normal FTS
成人搜索索引：adult FTS
普通操作日志：normal operations
成人操作日志：adult operations
```

普通模式启动时：

- 不打开 adult.db。
- 不扫描成人目录。
- 不加载成人封面缓存。
- 不显示 adult.* TAG。
- 不显示成人搜索结果。
- 不显示成人最近浏览。
- 不把成人库计入全局统计。

进入成人模式：

```text
输入软件内密码
-> 验证密码
-> 加载 adult.db
-> 加载成人 TAG 树和封面缓存
-> 成人库搜索和浏览可用
```

退出成人模式：

```text
关闭成人库页面
-> 清空当前成人搜索结果
-> 清空成人详情页状态
-> 卸载 adult.db 连接
-> 清理内存中的成人封面引用
```

密码策略：

- 首次启用成人库时设置密码。
- 密码只保存加盐哈希，不保存明文。
- 可设置自动锁定时间，例如 5、15、30 分钟。
- 软件关闭后默认锁定。
- 密码忘记时不显示原密码，只能通过恢复密钥或重置成人库配置处理。

限制说明：软件能保证普通模式内不可见成人库内容，但不能替代 Windows 账户权限、磁盘加密或文件系统级家长控制。如果孩子直接通过 Windows 文件管理器访问成人目录，软件无法阻止。

## 文件操作与整理

第一版采用“建议 + 手动确认”整理模式。

支持操作：

- 用默认播放器打开。
- 打开所在目录。
- 复制到指定目录。
- 剪切/移动到指定目录。
- 重命名文件。
- 批量移动。
- 批量重命名。
- 标记为忽略。
- 从库中移除但不删除文件。
- 删除到回收站。

第一版不做永久删除。

整理建议示例：

```text
D:\Movies\苏联电影\战争\片名 (年份)\
D:\Movies\导演\Akira Kurosawa\片名 (年份)\
X:\Adult\Studio\Series\Code\
```

批量整理流程：

```text
选择影片
-> 软件生成目标路径建议
-> 显示原路径、目标路径、冲突项
-> 用户确认
-> 执行复制、移动或重命名
-> 更新数据库路径
-> 写入操作日志
```

冲突处理：

- 跳过。
- 自动改名，例如 `片名 (年份) - 2.mkv`。
- 覆盖，默认不推荐且需要额外确认。
- 合并同一影片多版本记录，例如 1080p、4K、导演剪辑版。

默认策略是不覆盖，提示用户选择。

操作日志字段：

```text
操作类型
操作时间
源路径
目标路径
文件大小
操作前影片 ID
操作后影片 ID
所属库
执行结果
错误信息
```

离线硬盘处理：

- 显示为离线。
- 禁止移动和重命名。
- 允许编辑资料和 TAG。
- 允许加入整理计划。
- 硬盘重新接入后提示执行待处理计划。

普通库和成人库之间移动必须二次确认，因为这会改变可见性边界。移动后需要迁移数据库记录、封面缓存、TAG 和搜索索引。

## 错误处理

软件必须把失败变成可恢复状态：

- 扫描失败：记录失败目录和原因，允许重试。
- 文件离线：保留索引，标记离线，禁止移动和重命名。
- 资料识别失败：进入待确认工作台，不阻塞入库。
- AI 拒绝或超时：保留本地分类流程，不影响使用。
- 文件移动失败：不更新数据库路径，写入操作日志。
- 数据库路径失效：提供重新定位文件功能。
- 重复文件：提示候选重复，不自动删除。

## 隐私边界

默认原则是本地优先、联网授权：

- 不默认联网扫描资料。
- 不默认上传截图、视频、音频。
- 成人库默认不向云端 AI 发送图片。
- AI 调用前显示将发送的文本字段。
- API Key 本地加密保存。
- 普通库和成人库 AI 配置分开。
- 成人库资料、TAG、封面、搜索历史不进入普通模式。

## 测试重点

第一版测试重点：

- 扫描多硬盘、移动硬盘、离线硬盘。
- 时长和体积阈值能过滤短视频、缓存和游戏过场。
- SQLite FTS 能搜索文件名、路径、TAG、导演、年代、国家。
- 层级 TAG 点击和组合筛选正确。
- 成人库锁定时普通模式完全不可见。
- 文件移动、重命名后数据库路径正确更新。
- AI 失败、资料源失败时仍可手动补充。
- 操作日志能追踪文件去向。
- 普通库和成人库之间移动必须二次确认。

## MVP 验收标准

第一版完成时应满足：

- 用户能添加普通扫描目录和成人扫描目录。
- 软件能扫描出符合收录规则的视频，并过滤明显短视频和缓存。
- 用户能按硬盘查看 D 盘、E 盘、移动硬盘和离线硬盘的影片。
- 用户能通过 TAG 树筛选影片，例如国家、年代、类型、导演、存储位置。
- 用户能在待确认工作台补充线索，并让 AI 基于文本生成 TAG 和整理建议。
- 成人库在未输入密码时，普通模式完全不显示成人内容、成人 TAG、成人封面和成人统计。
- 用户能用默认播放器打开影片。
- 用户能手动确认后移动、复制、重命名文件，且数据库路径同步更新。
- 文件操作失败时有日志，不造成索引静默损坏。
