## 用户最新反馈

用户明确指出当前版本不能算可用产品：

1. 每次打开都要重新扫描，没有建立可复用索引。
2. 左侧按钮/分类不是真分类。
3. 没有分析影片，也没有根据分析结果分类。
4. 没有配置大模型进行分析的功能。
5. 视频展示丑，没有缩略图，也不能切换排版。

这些反馈成立。当前版本只是 MVP foundation，不是用户想要的可用影视索引软件。

## 已经完成的真实能力

### 项目基础

- `.NET 8` solution 已建立。
- WPF 桌面项目已建立。
- SQLite 初始化已建立。
- xUnit 测试项目已建立。
- BAT 启动脚本已建立：`run-movie-navigator.bat`

### 本地扫描

- 用户可以选择目录快速扫描。
- 当前快速扫描收录常见视频扩展名。
- 当前快速扫描默认过滤小于 `100MB` 的视频。
- 扫描结果会写入 SQLite 的 `media_items`、`media_search`、`media_tags`。

### 搜索与基础筛选

- 搜索框可以按标题、路径、硬盘、TAG 过滤当前内存列表。
- 扫描后会自动选中第一条媒体。
- 右侧详情会显示选中媒体的标题、路径、时长、分辨率、TAG。
- 左侧硬盘列表可以过滤当前扫描结果。
- 左侧 TAG 列表可以过滤当前扫描结果。
- 点击首页/普通库会清除筛选。

### 文件打开

- `默认播放器打开` 可以打开选中文件。
- `打开所在目录` 可以用 Explorer 定位选中文件。

### 安全基础

- 已有成人库状态模型。
- 已有密码哈希模型。
- 但成人库 UI 和实际解锁流程没有完成。

### 多语言基础

- 已有 `zh-CN` / `en-US` 资源文件。
- 当前 UI 仍有大量硬编码中文，需要后续继续本地化。

## 当前缺口

### P0 缺口：没有持久索引体验

数据库已有数据，但 App 启动时没有调用 repository 读取历史数据。

结果：
- 用户每次打开都看不到上次扫描结果。
- 用户感觉必须重新扫描。

需要：
- `IMediaRepository.GetAllAsync`
- `MainWindowViewModel.LoadIndexAsync`
- Window 启动后自动加载 SQLite
- 保存扫描目录
- 增量扫描全部扫描目录

### P0 缺口：没有真正分类系统

当前左侧不是分类系统，只是几个静态导航项和简单列表。

需要：
- 从数据库媒体生成分类 Facet。
- 分类包括硬盘、状态、文件类型、年代、分辨率、时长、TAG。
- 点击分类后中间列表必须过滤。
- 分类项必须显示数量。

### P0 缺口：没有缩略图和展示模式

当前中间区域只是文本列表，展示效果差。

需要：
- 数据库增加 `thumbnail_path`。
- 扫描时用 `ffmpeg` 生成本地 jpg 缩略图。
- UI 默认显示缩略图卡片。
- 支持切换：缩略图网格、紧凑列表、详情列表。
- 没有缩略图时显示设计过的占位卡片。

### P1 缺口：没有影片分析闭环

已有 `FfprobeVideoInspector`，但快速扫描 UI 没有接入。

需要：
- 扫描时读取时长、分辨率。
- 从文件名解析年份、番号、标题候选。
- 分析失败时进入待补充状态。
- 重新识别按钮要实际执行本地分析。

### P1 缺口：没有 AI 设置和文本分类

当前只有文案提到 AI，没有配置、没有调用、没有建议。

需要：
- 设置页配置 Provider、Base URL、Model、API Key。
- AI 默认关闭。
- 只发送文本线索，不发送截图、视频、音频。
- 调用前显示将发送的字段并要求用户确认。
- AI 输出只作为建议，用户确认后写入 TAG/标题/摘要。

### P2 缺口：成人库隔离还没有产品化

已有基础模型，但没有完整 UI。

需要：
- 成人库单独数据库或单独可见性策略。
- 成人库默认锁定。
- 未授权时不显示成人文件名、缩略图、TAG、路径。
- 成人库 AI 设置与普通库分开。

## 不应该继续做的事

- 不应该继续修一些“点了弹提示”的假功能。
- 不应该把静态标签叫分类。
- 不应该在没有持久索引前做复杂 AI。
- 不应该在没有缩略图前继续扩展文字列表。
- 不应该声称完成了分析/分类/成人库，只能说预留了基础。

## 下一阶段推荐执行顺序

### 阶段 1：持久索引和增量扫描

目标：
- 打开软件即可看到上次扫描结果。
- 保存扫描目录。
- 一键增量扫描全部目录。

验收：
- 第一次扫描 D 盘目录后关闭软件。
- 再次打开软件，不扫描也能看到之前结果。
- 新增一个视频，点增量扫描后出现。
- 删除或拔盘后，旧条目标记为离线/缺失，而不是直接消失。

### 阶段 2：真实分类

目标：
- 左侧分类来自数据库真实媒体。
- 分类项显示数量。
- 点击分类过滤结果。

验收：
- 点击 `硬盘/D:` 只显示 D 盘视频。
- 点击 `类型/mkv` 只显示 MKV。
- 点击 `清晰度/1080p` 只显示对应分辨率。
- 点击某个 TAG 只显示命中 TAG 的影片。

### 阶段 3：缩略图 UI

目标：
- 视频用缩略图卡片展示。
- 支持切换排版方式。

验收：
- 扫描后能看到缩略图或明确占位图。
- 可切换缩略图网格、紧凑列表、详情列表。
- 选中卡片后右侧详情正确更新。

### 阶段 4：本地影片分析

目标：
- 接入 `ffprobe`。
- 文件名解析标题/年份/番号候选。
- 重新识别按钮可实际分析。

验收：
- 时长和分辨率不再一直显示待分析。
- 文件名中有年份时能识别年份。
- 识别不了的进入待补充。

### 阶段 5：AI 文本分类助手

目标：
- 可配置 OpenAI-compatible API。
- 只发送文本线索。
- AI 返回分类建议，用户确认后写入。

验收：
- 设置页能保存 API 配置。
- 点击 AI 建议前能看到将发送的文本字段。
- AI 失败不会影响本地索引。
- 成人库默认不发送截图、视频、音频。
